Developer Handoff Package

Folder structure — Atomic Design
src/
├─ styles/
│  ├─ tokens.css        ← from handoff/
│  └─ globals.css
├─ lib/
│  ├─ tokens.ts         ← from handoff/
│  ├─ cn.ts             (clsx + tailwind-merge)
│  └─ format.ts         (Kz, dates pt-AO)
├─ components/
│  ├─ ui/               (atoms: Button, Input,
│  │                     Badge, Avatar, Spinner…)
│  ├─ composed/         (molecules: Field, Card,
│  │                     Alert, Tabs, Pagination…)
│  ├─ blocks/           (organisms: DataTable,
│  │                     Sidebar, KpiCard, Chart…)
│  └─ layouts/          (AppShell, MarketingPage)
└─ app/                 (routes)
Variant system — CVA pattern
const button = cva(
  'inline-flex items-center gap-2 font-sans
   font-semibold rounded-md transition
   duration-150 ease-standard
   focus-visible:ring-2
   focus-visible:ring-accent-500
   focus-visible:ring-offset-2
   disabled:opacity-45',
  { variants: {
      variant: {
        primary: 'bg-primary-700 text-white
                  hover:bg-primary-800
                  dark:bg-primary-400
                  dark:text-neutral-950',
        secondary: 'bg-primary-50
                  text-primary-700',
        outline: 'border border-neutral-200',
        ghost: '', danger: 'bg-danger-600…',
      },
      size: { sm:'h-8 px-3 text-[12.5px]',
              md:'h-10 px-4 text-sm',
              lg:'h-12 px-6 text-[15px]' },
    },
    defaultVariants: { variant:'primary',
                       size:'md' } });
Naming: PascalCase components, camelCase props, kebab-case files. One component per file; variants via CVA, never boolean soup.
